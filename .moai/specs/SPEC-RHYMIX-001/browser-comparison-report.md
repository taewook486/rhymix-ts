# 브라우저 비교 분석 보고서: ASIS (Rhymix PHP) vs TOBE (Rhymix-TS)

> 작성일: 2026-03-01
> 대상 시스템:
> - ASIS: http://localhost/ (Rhymix PHP)
> - TOBE: http://localhost:3000/ (Rhymix-TS, Next.js)

---

## 1. 개요

이 보고서는 기존 Rhymix PHP 시스템(ASIS)과 새로운 TypeScript/Next.js 기반 시스템(TOBE)의 브라우저 기반 비교 분석 결과를 담고 있습니다.

---

## 2. 기술 스택 비교

### ASIS (Rhymix PHP)

| 구분 | 기술 |
|------|------|
| 프레임워크 | Rhymix (PHP 기반 CMS) |
| 프론트엔드 | jQuery 2.2.4 |
| CSS | SCSS, XEICON |
| 테마 | XEDITION 레이아웃 |
| 슬라이더 | Swiper.js |
| 라우팅 | 쿼리 파라미터 기반 (?mid=xxx) |
| CSRF | 토큰 기반 보안 |

### TOBE (Rhymix-TS)

| 구분 | 기술 |
|------|------|
| 프레임워크 | Next.js 15 (React 19) |
| 프론트엔드 | React Server Components |
| CSS | Tailwind CSS, Radix UI |
| UI 컴포넌트 | shadcn/ui (Card, Button) |
| 인증 | Supabase Auth |
| 라우팅 | 파일 시스템 기반 (/ko/xxx) |
| 개발 도구 | Turbopack |

│ LOGIN WIDGET:                                                │
│   - 게스트 환영 메시지                                       │

| 네비게이션 메뉴 | 4개 메뉴 활성 | 비어있음 | 미구현 |
| 최근 게시물 | 표시됨 | 미표시 | 미구현 |
| 공지사항 | 표시됨 | 미표시 | 미구현 |

### 3.3 문서 페이지

## 4. 네비게이션 구조 비교

### ASIS 네비게이션 (쿼리 파라미터 기반)

```
/                    → 홈페이지
/?mid=board         → 자유게시판
/?mid=qna           → Q&A
/?mid=notice        → 공지사항
/?mid=member        → 회원 (로그인 페이지로 리다이렉트)
/?module=admin      → 관리자 (로그인 필요)
/board/login        → 로그인
/board/signup       → 회원가입
/board/write        → 글쓰기
```

### TOBE 네비게이션 (파일 시스템 기반)

```
/                    → 홈페이지
/ko/boards           → 게시판 목록 (404)
/ko/documents        → 문서 목록
/ko/members/login    → 로그인 (404)
/ko/members/profile  → 프로필
/ko/members/settings → 설정
/ko/admin            → 관리자 (로그인 리다이렉트)
/ko/signin           → 로그인 페이지
```

## 6. 미구현 기능 목록

### 높은 우선순위 (Critical)

1. **홈페이지 비주얼 슬라이더** - ASIS의 주요 시각 요소
2. **네비게이션 메뉴** - 현재 비어있음
3. **게시판 목록 페이지** - 404 에러
4. **로그인 페이지** - 404 에러 (/ko/members/login)

### 중간 우선순위 (High)

5. **웰컴 가이드 섹션** - 사이트 소개 콘텐츠
6. **최근 게시물 표시** - 홈페이지 위젯
7. **공지사항 표시** - 홈페이지 위젯
8. **게시판 글쓰기** - 현재 미작동

### 낮은 우선순위 (Medium)

9. **태그 기능** - 게시판 태그 시스템
10. **ID/PW 찾기** - 계정 복구 기능
11. **XEICON 통합** - 아이콘 시스템
12. **서브 헤더 배경** - 페이지 타이틀 배경 이미지

## 8. 권장 사항

### 8.1 즉시 조치 필요

1. **게시판 라우트 수정**: `/ko/boards` 페이지 404 해결
2. **로그인 라우트 일치**: `/ko/members/login` → `/ko/signin` 리다이렉트 추가
3. **네비게이션 메뉴 활성화**: 헤더에 메뉴 항목 추가

### 8.2 단기 개선 사항

1. **홈페이지 콘텐츠 추가**:
   - 비주얼 슬라이더 또는 히어로 이미지
   - 웰컴 가이드 섹션
   - 최근 게시물 위젯

2. **게시판 기능 완성**:
   - 글 목록 테이블
   - 글쓰기 폼
   - 검색 기능
   - 태그 시스템

### 8.3 장기 개선 사항

1. **디자인 시스템 정립**:
   - ASIS XEDITION 테마와 유사한 스타일 가이드
   - 일관된 컬러 팔레트

2. **접근성 강화**:
   - 스킵 링크 추가
   - ARIA 속성 보강
   - 키보드 네비게이션

3. **SEO 최적화**:
   - 메타 태그 보강
   - 구조화된 데이터

## 9. 결론

TOBE (Rhymix-TS) 시스템은 현대적인 기술 스택(Next.js, React, Supabase)을 기반으로 하여 확장성과 유지보수성 면에서 우수하지만, 현재 ASIS 시스템 대비 많은 기능이 미구현 상태입니다.

**구현 진척도 추정:**
- 기본 구조: 40%
- 페이지 라우팅: 30%
- UI 컴포넌트: 50%
- 비즈니스 로직: 20%

ASIS 시스템의 핵심 기능(게시판, 로그인, 네비게이션)을 우선적으로 구현하고, 이후 시각적 요소와 부가 기능을 단계적으로 추가할 것을 권장합니다.
